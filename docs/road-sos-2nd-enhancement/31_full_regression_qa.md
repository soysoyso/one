# STEP 31. 전체 기능 회귀 QA

## 목적

신규 기능과 기존 핵심 기능이 함께 동작하는지 Playwright로 통합 점검한다.

## 실행 스크립트

- 전체 회귀 QA: `node qa/road_sos_full_regression_qa.js`
- 화면 표시 QA: `node qa/road_sos_visible_qa.js`
- 기존 중간 QA: `node qa/road_sos_mid_qa.js`

## QA 중 발견 및 수정

1. 보고서 PDF 판정 보정
   - DOCX/HWPX는 ZIP 패키지 시그니처로 검증한다.
   - PDF는 `%PDF` 시그니처로 별도 검증하도록 QA 스크립트를 보정했다.

2. 체크리스트 저장 QA 요청 형식 보정
   - 체크리스트 항목처럼 동일 name이 반복되는 폼 필드는 `URLSearchParams`로 전송하도록 정리했다.
   - 실제 화면의 `serialize()` 전송 방식과 동일한 형태로 맞췄다.

3. 신규 기능 화면 한글 정리
   - 알림톡 수신자 설정
   - 일상점검 체크리스트 설정
   - 관리자 일상점검 조회/상세
   - 현장 사용자 일상점검 작성
   - 상황일지 관리
