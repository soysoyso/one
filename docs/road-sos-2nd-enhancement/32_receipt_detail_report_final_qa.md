# 접수상세-보고서 최종 QA

## 범위
- 현장관리 접수상세 필수 입력 항목 노출 확인
- 접수상세 저장 API 확인
- 저장 데이터 상세조회 확인
- 유지관리 결과보고서 DOCX/HWPX 생성 데이터 반영 확인
- 기존 주요 기능 회귀 QA 확인

## 확인 항목
- 문서번호, 차선/위치, 보고서 비고, 실작업량, 환산작업량, 기성작업량 필드가 현장관리 접수상세 화면에 노출된다.
- 작업정보 장비/인력/자재/범위 입력 영역이 현장관리 접수상세 화면에 노출된다.
- 접수상세 저장 시 본문 필드와 작업정보 4종이 DB에 저장된다.
- 저장된 접수는 현장관리 목록 키워드 검색으로 조회된다.
- 저장된 접수의 차선/비고/상세위치/작업량이 유지관리 결과보고서 DOCX/HWPX에 반영된다.

## 수정 사항
- 로컬 DB 마이그레이션에 포트홀 필수 컬럼, 사진 호환 컬럼, 작업정보 상세 컬럼을 보강했다.
- 현장관리 목록 조회 서비스에서 `keyword` 파라미터를 mapper 조건으로 전달하도록 수정했다.
- 유지관리 결과보고서 공통 행 출력에 상세정보를 포함하도록 수정했다.
- `local` Spring profile에서도 콘솔 로그가 출력되도록 logback 설정을 보강했다.
- Playwright QA에 접수상세 저장부터 DOCX/HWPX 문서 내용 확인까지 이어지는 전용 시나리오를 추가했다.

## 원인
- 기존 로컬 DB 볼륨에는 `pothole_photo`와 작업정보 테이블 일부가 예전 컬럼 구조로 남아 있었다.
- 저장 이후 대표사진 보정 로직과 작업정보 저장 로직이 신규 컬럼명을 사용하면서 SQL 오류가 발생했고, 트랜잭션이 롤백되었다.
- 목록 mapper에는 키워드 조건이 있었지만 서비스가 `keyword`를 전달하지 않아 신규 접수 검색 검증이 실패했다.
- 보고서 생성은 성공했지만 상세위치 값이 문서 행에 표시되지 않아 필수 데이터 반영 검증이 부족했다.
