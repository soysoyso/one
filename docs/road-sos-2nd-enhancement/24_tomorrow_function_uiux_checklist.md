# 내일 기능/UIUX 확인 체크리스트

## 1. 로컬 환경

- Docker Desktop 실행
- `road-sos`, `road-sos-mysql` 컨테이너 실행 상태 확인
- 접속 URL: `http://localhost:8703`
- 관리자 계정: `admin / admin123`
- 현장 사용자 계정: `field / field123`

## 2. 관리자 확인 순서

### 2.1 현장관리/접수

- URL: `/admin/ims/dashboard`
- 접수 목록이 정상 표시되는지 확인
- 접수 상세 모달에서 기존 필드와 추가 보고서 필드가 어색하지 않은지 확인
- 접수 선택 후 보고서 다운로드 형식 선택 확인
  - PDF
  - DOCX
  - HWPX
- 이미지 다운로드 버튼과 보고서 다운로드 버튼의 구분이 명확한지 확인

### 2.2 알림톡 수신자 설정

- URL: `/admin/notification/recipients`
- 수신자 목록/검색/추가/수정/삭제 흐름 확인
- 알림 유형 선택값 확인
  - 포트홀 접수
  - 포트홀 처리완료
  - 일상점검
  - 상황일지
- 로컬 발송은 실제 발송이 아니라 서버 로그의 스텁 로그로 확인한다.

### 2.3 일상점검 체크리스트 설정

- URL: `/admin/daily-checklists/setting`
- 체크리스트 추가/수정/삭제 확인
- 항목 유형 확인
  - 체크
  - 텍스트
  - 숫자
  - 긴 문장
- 필수 항목 표시와 정렬 순서가 운영자가 이해하기 쉬운지 확인

### 2.4 일상점검 조회

- URL: `/admin/daily-checks`
- 목록 검색/상세 확인
- 점검 항목 응답 표시 확인
- 점검 전/후 사진 미리보기 확인
- DOCX/HWPX 내보내기 버튼 확인

### 2.5 상황일지

- URL: `/admin/situation-logs`
- 목록/등록/수정/삭제 확인
- 주간/야간 선택 흐름 확인
- 필수값 누락 시 안내가 자연스러운지 확인
- DOCX/HWPX 내보내기 버튼 확인

## 3. 현장 사용자 확인 순서

### 3.1 일상점검 작성

- URL: `/manage/daily-checks/form`
- 모바일 화면 폭에서 입력이 편한지 확인
- 체크리스트 항목이 자연스럽게 표시되는지 확인
- 필수값 누락 안내 확인
- 점검 전/후 사진 첨부 확인
- 저장 후 관리자 화면에서 조회되는지 확인

## 4. 문서 출력 확인

- 포트홀 관리대장 DOCX 다운로드
- 포트홀 관리대장 HWPX 다운로드
- 일상점검 DOCX/HWPX 다운로드
- 상황일지 DOCX/HWPX 다운로드
- HWPX는 PoC 수준이므로 한글 오피스에서 열리는지 별도 확인한다.

## 5. UIUX 확인 포인트

- 한글 깨짐 여부
- 버튼명과 실제 동작이 맞는지
- 관리자 화면의 버튼/검색/테이블 간격이 기존 road-sos 톤과 맞는지
- 모바일 현장 화면에서 스크롤과 입력 흐름이 불편하지 않은지
- 오류 메시지가 `500`처럼 개발자용으로 보이지 않고 사용자가 이해할 수 있는지

## 6. 내일 발견 시 바로 개선할 후보

- 버튼명/위치/간격 조정
- 필수값 안내 문구 정리
- 보고서 파일명 규칙 정리
- HWPX 호환성 보강
- 접수 상세 보고서 필드 추가 보강
