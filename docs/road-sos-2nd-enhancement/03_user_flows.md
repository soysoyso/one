# 사용자 흐름 설계

## 1. 공통 진입

| 사용자 | 진입 경로 | 인증 |
|---|---|---|
| 현장 사용자 | `/ims` 또는 현장 모바일 메뉴 | 기존 현장 사용자 인증 |
| 관리자 | `/admin` | 관리자 로그인 |
| 보고서 담당자 | 관리자 현장관리/보고서 메뉴 | 관리자 권한 |

## 2. 보고서 출력 흐름

### 단건 보고서

1. 관리자가 현장관리 목록 또는 상세 화면에 진입한다.
2. 접수 건을 선택한다.
3. `미리보기` 또는 `다운로드`를 선택한다.
4. 보고서 템플릿을 선택한다.
5. 출력 형식을 선택한다: PDF, DOCX, 향후 HWPX.
6. 시스템이 해당 템플릿에 필요한 데이터를 조회한다.
7. 문서를 생성하여 브라우저에 반환한다.

### 복합 보고서

1. 관리자가 접수 목록에서 여러 건을 체크한다.
2. `복합 보고서 생성`을 선택한다.
3. 템플릿을 선택한다.
4. 기간/유형/상태 기준이 필요한 템플릿은 추가 조건을 입력한다.
5. PDF 또는 DOCX로 다운로드한다.

## 3. 일상점검 흐름

### 관리자 체크리스트 관리

1. 관리자가 `관리자 설정 > 일상점검 체크리스트`에 진입한다.
2. 현장/도로/공통 여부를 선택한다.
3. 점검 항목을 등록한다.
4. 항목 순서, 필수 여부, 사용 여부를 설정한다.
5. 저장 후 사용자 화면에 반영된다.

### 사용자 일상점검 작성

1. 사용자가 `일상점검` 메뉴에 진입한다.
2. 날짜와 현장을 선택한다.
3. 체크리스트 항목을 확인한다.
4. 각 항목의 상태와 점검내용을 입력한다.
5. 작업 전 이미지를 등록한다.
6. 필요 시 조치내용과 작업 후 이미지를 등록한다.
7. 저장 또는 완료 처리한다.

### 일상점검 보고서 출력

1. 관리자가 일상점검 목록에서 기간/현장으로 검색한다.
2. 점검일지를 선택한다.
3. `일상점검일지` 또는 `일상점검 결과보고서` 템플릿을 선택한다.
4. PDF/DOCX로 출력한다.

## 4. 상황일지 흐름

1. 관리자 또는 권한 있는 사용자가 `상황일지` 메뉴에 진입한다.
2. 날짜와 주/야간 구분을 선택한다.
3. 상황 발생 시간을 입력한다.
4. 상황 내용을 입력한다.
5. 저장하면 리스트에 시간순으로 표시된다.
6. 필요 시 수정/삭제한다.
7. 기간 또는 일자 기준으로 상황일지 보고서를 출력한다.

## 5. 알림톡 수신자 설정 흐름

1. 관리자가 `관리자 설정 > 알림톡 수신자`에 진입한다.
2. 알림 유형을 선택한다.
3. 수신자 목록을 등록/수정/삭제한다.
4. 수신 여부와 우선순위를 설정한다.
5. 실제 발송 연동 전까지는 로컬에서 발송 대상 조회/저장까지만 검증한다.

## 6. 실패/예외 화면

| 상황 | 사용자 메시지 |
|---|---|
| 필수값 누락 | 필수 항목을 입력해주세요. |
| 권한 없음 | 해당 메뉴에 접근할 권한이 없습니다. |
| 템플릿 없음 | 선택한 보고서 양식을 찾을 수 없습니다. |
| 문서 생성 실패 | 보고서 생성 중 오류가 발생했습니다. |
| 이미지 없음 | 등록된 대표 이미지가 없습니다. |
