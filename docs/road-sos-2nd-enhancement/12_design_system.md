# road-sos 공통 디자인 기준

## 1. 목적

2차 고도화에서 새로 추가되는 알림톡 수신자, 일상점검, 상황일지, 보고서 설정 화면의 UI를 기존 road-sos 화면과 일관되게 유지한다.

## 2. 현재 디자인 기준

### 공통 폰트

- `NotoSansKR`
- 관리자 기본 크기: 15px
- 현장/모바일 기본 크기: 16px

### 주요 색상

| 용도 | 색상 |
|---|---|
| 헤더/주요 네비게이션 | `#00263e` |
| 활성 메뉴 강조 | `#31e8fd` |
| 기본 배경 | `#f9fafb` |
| 모달 내부 배경 | `#f5f5f5` |
| 테이블 헤더 | `#6c757d` |
| 기본 파란 버튼 | Bootstrap `#0d6efd` |
| 위험/삭제 | Bootstrap `#dc3545` |
| 성공/완료 | Bootstrap `#198754` |

### 레이아웃

관리자 화면:

- 상단 네이비 헤더
- 콘텐츠 영역은 좌우 여백이 있는 업무형 레이아웃
- 검색 조건은 상단 박스 영역에 배치
- 목록은 테이블 중심
- 상세/등록은 대형 모달과 아코디언 사용

현장 사용자 화면:

- 모바일 우선
- 상단 고정 헤더
- 카드형 리스트
- 큰 입력 컨트롤과 넓은 터치 영역
- 사진 업로드/미리보기 중심

## 3. 컴포넌트 기준

### 버튼

- 저장/수정: `btn btn-primary`
- 삭제: `btn btn-danger` 또는 텍스트형 삭제 버튼
- 보조 액션: `btn btn-outline-secondary`, `btn btn-outline-primary`
- 관리자 테이블 안 버튼은 작고 간결하게 유지한다.

### 입력 필드

- Bootstrap `form-control`, `form-select` 사용
- 라벨은 `form-label fw-semibold`
- 필수값은 기존 방식처럼 라벨 앞에 `*` 강조 사용
- 날짜/시간은 기존 flatpickr 스타일을 따른다.

### 테이블

- 관리자 목록은 테이블 중심
- 헤더는 회색 배경, 흰색 텍스트
- 셀 정렬은 중앙 정렬을 기본으로 하되, 긴 텍스트는 좌측 정렬 허용
- 반복 입력 테이블은 행 추가/삭제 버튼을 제공한다.

### 모달

- 기존 `#ims-add-modal` 구조를 따른다.
- 모달 헤더는 네이비 배경
- 본문은 연회색 배경
- 입력 그룹은 흰색 박스 또는 아코디언으로 구분
- 접수상세 확장 필드는 기존 모달에 새 아코디언 섹션으로 추가한다.

### 아코디언

- 관리자 상세 입력 영역은 아코디언을 기본으로 한다.
- 섹션명은 업무 명칭으로 짧게 작성한다.
- 예: 기본정보, 위치정보, 작업정보, 보고서 추가정보, 사진대지

## 4. 신규 화면별 디자인 지침

### 알림톡 수신자 설정

- 관리자 설정 화면 톤을 따른다.
- 상단 검색/필터: 알림유형, 현장, 사용여부
- 목록: 수신자명, 전화번호, 현장, 알림유형, 사용여부, 관리
- 등록/수정은 모달 사용

### 일상점검 관리자

- 체크리스트 목록과 항목 편집 영역을 분리한다.
- 항목은 순서 변경과 사용여부를 지원한다.
- 항목 추가는 테이블 행 추가 방식이 적합하다.

### 일상점검 사용자

- 모바일 카드형 입력을 우선한다.
- 체크리스트 항목은 한 화면에서 빠르게 확인 가능해야 한다.
- 사진 등록은 기존 현장 사진 업로드 패턴을 재사용한다.

### 상황일지

- 관리자 업무형 목록 UI를 따른다.
- 날짜, 주/야간, 현장 필터를 상단에 둔다.
- 상황 입력은 시간과 내용을 빠르게 추가하는 구조가 적합하다.

### 보고서 추가정보

- 어드민 접수상세 모달 내부에 새 아코디언으로 배치한다.
- 기존 접수/작업 데이터와 중복 입력하지 않는다.
- 보고서에만 필요한 보조값은 선택 입력으로 시작한다.

## 5. 금지/주의

- 새 화면마다 다른 색상 체계를 만들지 않는다.
- 관리자 화면에 과한 카드형 랜딩 UI를 만들지 않는다.
- 기존 접수상세의 저장 흐름을 깨지 않는다.
- 보고서 필드 때문에 필수 접수 저장이 막히지 않도록 한다.
- HWPX/DOCX 출력 UI는 마지막 단계 전까지 과하게 노출하지 않는다.
