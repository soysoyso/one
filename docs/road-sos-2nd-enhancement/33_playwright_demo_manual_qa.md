# Playwright 시연 QA 매뉴얼

## 목적
브라우저를 띄운 상태에서 운영자가 따라 볼 수 있는 방식으로 2차 고도화 기능을 검증한다.

## 시나리오 A. 현장관리 접수 등록부터 보고서 출력
1. 관리자 로그인
2. 현장관리 진입
3. `접수 등록` 선택
4. 공통 필수항목 입력
   - 문서번호
   - 현장
   - 작업유형
   - 작업상태
   - 접수자
   - 날씨
   - 기온
   - 접수일시
5. 위치정보 입력
   - 주소
   - STA 정보
   - 위치상세정보
6. 접수/작업 내용 입력
   - 접수내용
   - 작업내용
7. 작업정보 입력
   - 투입 장비
   - 투입 인력
   - 투입 자재
   - 작업 범위
8. 보고서 추가정보 입력
   - 차선/위치 보조정보
   - 실작업량
   - 환산작업량계
   - 작업량계상
   - 보고서 비고
9. 저장 확인
10. 저장된 접수 상세 재조회
11. 유지관리 결과보고서 DOCX 다운로드
12. DOCX/HWPX 내부 데이터 반영 검증

## 시나리오 B. 추가 기능 흐름
1. 알림톡 수신자 설정
   - 수신자 추가, 검색, 전화번호 마스킹을 확인한다.
2. 일상점검 체크리스트 설정
   - 점검 양식과 필수 항목을 구성하는 화면을 확인한다.
3. 현장 사용자 일상점검 작성
   - 모바일 현장 사용자 화면에서 점검 작성 진입을 확인한다.
4. 관리자 일상점검 조회
   - 관리자 화면에서 작성된 점검을 조회하고 DOCX/HWPX 출력 버튼을 확인한다.
5. 상황일지 관리
   - 상황 등록, 상세 조회, DOCX/HWPX 출력 버튼을 확인한다.
6. 보고서 템플릿 출력 제어
   - 포트홀 관리대장은 PDF/DOCX/HWPX를 지원한다.
   - 그 외 보고서 템플릿은 PDF를 비활성화하고 DOCX/HWPX 중심으로 출력한다.

## 실행 명령
```powershell
node qa/road_sos_demo_manual_qa.js
```

## 운영 메모
- 시연 스크립트는 기본적으로 QA 접수 데이터를 로컬 DB에 남긴다.
- 정리까지 자동으로 하려면 `KEEP_DEMO_DATA=0` 환경변수를 지정한다.
- 브라우저가 닫히기 전 대기시간은 `DEMO_QA_HOLD_MS`로 조정한다.
