# road-sos 2차 고도화 서비스 정의

## 1. 서비스명

YIDO 인프라 운영관리 시스템 2차 고도화

## 2. 문제 정의

현재 시스템은 사고접수, 현장관리, 포트홀/도로파손 접수 및 PDF 보고서 출력 중심으로 구성되어 있다. 2차 고도화에서는 현장 운영자가 반복적으로 수행하는 일상점검, 상황기록, 작업일보, 복합 보고서 생성을 시스템화해야 한다.

핵심 문제는 다음과 같다.

- 보고서 양식이 여러 종류로 분리되어 있으나 시스템 내 템플릿 선택/출력 구조가 아직 충분하지 않다.
- 현재 보고서는 PDF 중심이며, 실제 업무에서는 편집 가능한 Word 문서가 필요하다.
- 향후 공공/현장 문서 호환을 위해 HWPX 출력 가능성도 검토해야 한다.
- 일상점검과 상황일지는 기존 접수/작업 프로세스와 유사하지만 별도 메뉴, 별도 데이터 구조가 필요하다.
- 발생장소, 공종, 투입자재, 투입인력, 투입장비 등 보고서 필드가 수기 입력 위주라 표준화가 필요하다.

## 3. 사용자 유형

| 사용자 | 역할 | 주요 기능 |
|---|---|---|
| 현장 사용자 | 접수/점검/작업 기록 | 일상점검 작성, 작업 전/후 이미지 등록, 상황 기록 |
| 관리자 | 운영 데이터 관리 | 체크리스트 항목 관리, 발생장소/공종 관리, 알림톡 수신자 관리 |
| 보고서 담당자 | 문서 출력/취합 | PDF/DOCX 보고서 다운로드, 복합 보고서 생성, 템플릿 선택 |
| QA 담당자 | 검증 | 부분 QA, 통합 QA, 오픈 전 테스트 요청 |

## 4. 핵심 기능

### 공통

- 알림톡 수신자 설정
- 도로/현장별 발생장소 마스터 관리
- 공종/작업유형/보고서 유형 관리

### 접수/보고서

- 보고서 DOCX 출력
- 보고서 PDF 기존 출력 유지
- HWPX 확장 가능 구조 검토
- 보고서 템플릿 6종 제작 및 적용
- 선택 접수 건 복합 보고서 생성
- 미리보기/다운로드 시 템플릿 선택

### 일상점검

- 사용자 일상점검 체크리스트 작성
- 관리자 체크리스트 항목 생성/수정/삭제
- 점검일지 및 점검 결과 보고서 출력
- 작업 전/작업 후 이미지 연결

### 상황일지

- 상황일지 별도 메뉴
- 주간/야간 구분
- 상황정보 입력: 시간, 내용
- 리스트 조회
- 입력/수정/삭제
- 보고서 출력

## 5. 기대효과

- 현장 기록과 보고서 작성의 중복 입력 감소
- 업무 양식별 출력 자동화
- 점검/작업/상황 이력의 조회 가능성 향상
- 보고서 편집 및 제출 대응력 향상
- QA 및 오픈 전 검증 범위 명확화

## 6. KPI

| 항목 | 측정 기준 |
|---|---|
| 보고서 생성 시간 | 수기 작성 대비 단축 여부 |
| 중복 입력 감소 | 접수/점검/보고서 간 동일 필드 재입력 감소 |
| 템플릿 적용률 | 요구 양식 6종 출력 가능 여부 |
| QA 통과율 | 부분 QA 및 통합 QA 결함 수 |
| 현장 사용성 | 일상점검 작성 완료율 |

## 7. MVP 범위

- 로컬 DB 기준 신규 테이블 설계 및 샘플 데이터 구성
- 보고서 템플릿 목록/선택 구조 설계
- DOCX 출력 PoC 및 1개 양식 우선 적용
- 일상점검 체크리스트 관리자/사용자 기본 CRUD
- 상황일지 기본 CRUD
- 부분 QA 시나리오 작성

## 8. 제외 범위

- 운영 DB 직접 변경
- 실제 알림톡 외부 발송 API 운영 연동
- 운영 배포 자동화
- HWPX 완전 구현

단, HWPX는 설계상 출력 타입으로 확장 가능하도록 인터페이스와 데이터 모델에서 고려한다.
