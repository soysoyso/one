# 34. 기획서 기준 재점검 감사

작성일: 2026-05-26

## 1. 점검 목적

최근 구현 결과가 기획 문서의 메뉴별 정책, 목적, 사용자 흐름과 다르게 체감되는 문제가 확인되었다.  
본 문서는 `docs/road-sos-2nd-enhancement` 하위 기획/정책/Step 문서를 기준으로 현재 구현 상태를 다시 대조하고, 누락 범위와 우선순위를 정리한다.

## 3. 총평

기획서의 큰 방향은 다음 4개 축이다.

1. 현장관리 접수 데이터를 기반으로 보고서 양식별 출력
2. 일상점검 체크리스트 등록, 사용자 점검 수행, 이력 조회, 보고서 출력
3. 상황일지 일자/주야간/시간대별 기록, 조회, 수정, 삭제, 보고서 출력
4. 알림톡 수신자 설정과 로컬 검증

현재 구현은 일부 API와 관리자 화면은 존재하지만, 사용자 화면과 실제 업무 흐름이 빠져 있거나, 버튼/패널/양식 구조가 기획 의도와 다르게 연결된 부분이 있었다.  
2026-05-26 현재 일부는 보정했으나, 아래의 남은 갭은 별도 완료 기준으로 추적해야 한다.

## 6. 가장 큰 누락

1. **사용자 상황일지 화면 부재**  
   기획 흐름은 "관리자 또는 권한 있는 사용자"가 상황일지를 입력/조회할 수 있게 되어 있으나 현재는 `/admin/situation-logs`만 있다.

2. **보고서 양식 원본 적합성 미검증**  
   2026-05-26 기준 템플릿별 구조는 분리했지만, 기획서가 말한 실제 업무 양식의 표, 결재란, 사진 배치, 문구와 1:1로 맞는지는 아직 아니다.

3. **일상점검 수정/이력 정책 미구현**  
   `DAILY-003`에 관리자 수정 가능성과 수정 이력 검토가 언급되어 있으나, 현재는 작성 후 조회 중심이다.

4. **공종/발생장소/작업유형/투입자원 마스터 관리 부재**  
   보고서 품질을 좌우하는 기본 마스터 관리가 아직 약하다.

5. **문서와 현재 정책 불일치**  
   문서에는 HWPX PoC/지원이 남아 있으나, 현재 최신 요구는 HWPX 제외다. 정책 문서를 최신 결정으로 갱신해야 한다.

## 7. 우선순위 제안

### P0. 즉시 정렬

- 사용자 상황일지 메뉴/목록/등록/상세 구현
- 일상점검 사용자 이력 화면 모바일 UI QA
- 보고서 템플릿별 실제 양식 샘플 기준 재매핑

### P1. 업무 완성도

- 일상점검 수정/관리자 보정/이력 정책 확정 및 구현
- 일상점검 결과보고서와 일지를 별도 양식으로 분리
- 상황일지 하루 단위 타임라인 UX 보강

### P2. 운영 관리

- 발생장소/공종/작업유형/투입자원 마스터 관리
- 보고서 템플릿 관리 메뉴
- 권한 경계 회귀 QA

## 8. 최신 변경 반영 메모

2026-05-26 현재 코드에 반영된 보정:

- 일상점검 체크리스트 답변 옵션 저장 컬럼 추가
- 사용자 일상점검 이력 목록/상세 화면 추가
- 상황일지/체크리스트/수신자 사이드패널 열림 오류 수정
