# 관리자/사용자 화면 설계 초안

## 1. 관리자 메뉴

| 메뉴 | 주요 기능 |
|---|---|
| 현장관리 > 보고서 | 템플릿 선택, PDF/DOCX 다운로드, 복합 보고서 생성 |
| 관리자 설정 > 보고서 템플릿 | 템플릿 목록, 사용 여부, 지원 형식 확인 |
| 관리자 설정 > 알림톡 수신자 | 알림 유형별 수신자 CRUD |
| 관리자 설정 > 일상점검 체크리스트 | 체크리스트 및 항목 CRUD |
| 관리자 설정 > 발생장소/공종 | 도로별 발생장소, 공종 마스터 관리 |
| 현장관리 > 일상점검 | 점검 목록 조회, 상세, 보고서 출력 |
| 현장관리 > 상황일지 | 상황일지 CRUD, 보고서 출력 |

## 2. 사용자 메뉴

| 메뉴 | 주요 기능 |
|---|---|
| 일상점검 | 체크리스트 작성, 사진 등록, 저장/완료 |
| 상황일지 | 권한 부여 시 상황 입력/조회 |
| 현장 접수 상세 | 기존 접수/작업 흐름 유지 |

## 3. 보고서 화면 변경

기존 목록의 PDF 다운로드 버튼은 다음 구조로 확장한다.

- `미리보기`
- `다운로드`
- 템플릿 선택 모달
- 출력 형식 선택: PDF, DOCX, 향후 HWPX

초기 구현에서는 기존 버튼을 제거하지 않고, 신규 버튼을 병행 추가하는 방식이 안전하다.

## 3-1. 어드민 접수상세 데이터 확장

보고서 양식 생성을 위해 어드민 접수상세 화면에 보고서용 데이터 입력 영역을 추가한다.

### 추가 권장 영역

| 영역 | 필드 |
|---|---|
| 보고서 기본정보 | 문서번호, 작업일자, 보고월, 담당자 |
| 위치 상세정보 | 행정구역, 도로이정, 방향, STA, 상세위치, 차선 |
| 발생/분류정보 | 공종, 작업유형, 발생장소, 포장형식, 발생수량 |
| 작업/조치정보 | 작업내용, 조치내용, 작업상태, 조치일시 |
| 투입자원 | 투입인력, 투입장비, 투입자재, 사용량, 단위, 잔량, 금액 |
| 작업량 | 실작업량, 환산작업량계, 작업량계상, 비고 |
| 사진대지 | 작업 전/후 대표사진, 사진 설명, 사진 순서 |

### 화면 원칙

- 기존 접수/작업 입력 흐름은 유지한다.
- 보고서용 추가 필드는 접수상세 내 별도 섹션 또는 탭으로 분리한다.
- 보고서 필드가 비어 있어도 접수 저장은 가능하되, 보고서 출력 시 누락 경고를 제공한다.
- 공종, 발생장소, 포장형식은 장기적으로 마스터 데이터 기반 선택형으로 전환한다.

## 4. 일상점검 작성 화면

필수 구성:

- 점검일자
- 현장/도로
- 체크리스트 항목
- 항목별 상태
- 점검내용
- 작업 전 사진
- 조치내용
- 작업 후 사진
- 저장/완료 버튼

## 5. 상황일지 화면

필수 구성:

- 일자
- 주/야간
- 시간
- 제목
- 내용
- 등록/수정/삭제
- 리스트
- 보고서 출력
