# Playwright 화면 표시 QA 시나리오

## 목적

기능 QA를 실행하기 전에 먼저 시나리오를 정의하고, Playwright가 실제 브라우저 화면을 띄운 상태에서 시나리오를 따라가며 확인한다.

## 실행 기준

- URL: `http://localhost:8703`
- 관리자 계정: `admin / admin123`
- 현장 사용자 계정: `field / field123`
- 실행 스크립트: `qa/road_sos_visible_qa.js`

## 시나리오 1. 관리자 로그인 및 현장관리 보고서 출력 UI

1. `/admin/login`으로 이동한다.
2. 관리자 계정으로 로그인한다.
3. `/admin/ims/dashboard`로 이동한다.
4. 현장관리 접수 목록 영역이 보이는지 확인한다.
5. 보고서 템플릿 선택 콤보박스가 보이는지 확인한다.
6. 출력 형식 `PDF/DOCX/HWPX` 선택 콤보박스가 보이는지 확인한다.
7. `사진대지` 템플릿을 선택했을 때 PDF 옵션이 비활성화되고 DOCX로 전환되는지 확인한다.

## 시나리오 2. 알림톡 수신자 목록 개인정보 표시

1. `/admin/notification/recipients`로 이동한다.
2. 알림톡 수신자 설정 화면이 보이는지 확인한다.
3. 테스트 수신자를 생성한다.
4. 테스트 수신자를 검색한다.
5. 목록의 휴대전화번호가 `010-****-1234` 형태로 마스킹되는지 확인한다.
6. 테스트 수신자를 삭제한다.

## 시나리오 3. 일상점검 관리자 화면

1. `/admin/daily-checklists/setting`으로 이동한다.
2. 체크리스트 설정 화면이 보이는지 확인한다.
3. `/admin/daily-checks`로 이동한다.
4. 일상점검 조회 화면과 DOCX/HWPX 출력 버튼이 보이는지 확인한다.

## 시나리오 4. 상황일지 화면

1. `/admin/situation-logs`로 이동한다.
2. 상황일지 목록 화면이 보이는지 확인한다.
3. DOCX/HWPX 출력 버튼이 보이는지 확인한다.

## 시나리오 5. 현장 사용자 일상점검 작성

1. 모바일 크기 브라우저 창에서 `/manage/login`으로 이동한다.
2. 현장 사용자 계정으로 로그인한다.
3. `/manage/daily-checks/form`으로 이동한다.
4. 모바일 폭에서 일상점검 작성 화면이 보이는지 확인한다.

## 판정 기준

- 브라우저가 실제로 열린 상태에서 각 화면을 눈으로 확인할 수 있어야 한다.
- 자동 검증 실패 시 콘솔에 실패 단계가 표시되어야 한다.
- QA가 끝난 뒤 브라우저를 잠시 유지해 마지막 화면을 확인할 수 있어야 한다.
